Проект: DevSafe GUI

Назначение: защита проектов от потери кода

1. Цель системы

Обеспечить автоматическое и непрерывное сохранение исходного кода проектов разработчика путём:

регулярных Git-коммитов,

автоматической отправки изменений в GitHub,

резервного копирования проекта в локальную папку на другом диске,

без необходимости ручных действий со стороны пользователя.

2. Область применения

ОС: Windows

Пользователь: разработчик / архитектор

Тип проектов:

Python

mixed (Python + configs + docs)

Система используется во время активной разработки

3. Основные ограничения
3.1 Количество проектов

система должна поддерживать до 5 проектов

одновременно активен только 1 проект

остальные проекты находятся в состоянии не отслеживаются

3.2 Безопасность

система НЕ ДОЛЖНА:

удалять файлы пользователя

коммитить .env, секреты, бинарники

изменять .gitignore

система ОБЯЗАНА:

уважать .gitignore

работать только в рамках выбранного проекта

4. Функциональные требования
4.1 Управление проектами

Система должна позволять:

добавить проект в отслеживание

удалить проект из отслеживания (без удаления файлов)

переключить активный проект

хранить список проектов между перезапусками приложения

Для каждого проекта пользователь указывает:

Параметр	Обязателен
Название проекта	✅
Путь к проекту	✅
GitHub репозиторий (remote)	✅
Ветка	✅
Папка резервного копирования	✅
4.2 Отслеживание изменений

Для активного проекта система должна:

реагировать на изменения файлов

не реагировать на изменения в .git, .venv, кешах

агрегировать изменения (не коммитить каждое сохранение)

4.3 Git-операции

При обнаружении изменений система должна:

выполнить git add (только разрешённые файлы)

выполнить git commit с авто-сообщением

выполнить git push в указанный GitHub-репозиторий и ветку

⚠️ Если push невозможен:

коммит всё равно должен быть создан

ошибка фиксируется в логе

4.4 Резервное копирование

После каждого коммита система должна:

синхронизировать проект в указанную папку резерва

резерв должен отражать текущее состояние проекта

.git каталог не копируется

5. Пользовательский интерфейс (GUI)

Приложение должно иметь графический интерфейс с:

окном управления проектами

списком добавленных проектов (до 5)

визуальным указанием активного проекта

кнопками:

Add project

Remove from tracking

Start

Stop

отображением текущего статуса:

IDLE / ACTIVE / ERROR

логом последних действий

6. Нефункциональные требования
6.1 Надёжность

приложение должно корректно работать при:

падении IDE

временном отсутствии интернета

ошибке GitHub

при перезапуске приложение восстанавливает состояние

6.2 Производительность

приложение не должно:

нагружать CPU в простое

создавать десятки коммитов в минуту

допускается задержка (debounce) перед коммитом

6.3 Удобство

пользователь не обязан:

помнить про коммиты

помнить про push

вручную делать бэкапы

7. Критерии приёмки (Definition of Done)

Система считается реализованной, если:

✅ можно добавить до 5 проектов

✅ можно выбрать активный проект

✅ при изменении файла:

создаётся коммит

изменения отправляются в GitHub

создаётся резервная копия

✅ удаление проекта не затрагивает файлы

✅ система переживает перезапуск

8. Явно вне scope (на данном этапе)

❌ шифрование бэкапов
❌ работа в облаке
❌ более 5 проектов
❌ мобильная версия
❌ CI/CD интеграции
